        Серверная часть крметики-нолики
    В серверной части используются: серверные сокеты для прослушивания входящих подключений
    и многопоточность(для каждого клиента свой поток)
    
    Серверная часть принимает ходы клиентов и отправляет их оппоненту, а также следит за
    стостоянием поля и определяет победителя и проигравшего (отпраляет инофрмацию клиентам)

                Описание классов сервера:
        TicTacToeServer.java
Логика работы:
1. Создать ServerSocket на указанном порту
2. Принимать входящие подключения в бесконечном цикле
3. Для каждого подключения создать ClientHandler
4. Запустить ClientHandler в отдельном потоке
5. Хранить все активные подключения

        ClientHandler.java
1. Инициализация потоков ввода/вывода
2. Добавление игрока в GameSession через GameSessionManager
3. Бесконечный цикл чтения сообщений от клиента
4. Парсинг и обработка каждого сообщения
5. Передача команд в GameSession
6. Отправка обновлений клиенту

        GameSession.java
1. Создание пустого игрового поля
2. Добавление двух игроков (X и O)
3. Прием ходов от игроков
4. Проверка валидности хода
5. Обновление состояния поля
6. Проверка на победу/ничью
7. Рассылка обновлений обоим игрокам

        Структура зависимостей:
ServerMain (независимый)
    ↓ использует
TicTacToeServer (зависит от всех ниже)
    ↓ содержит
ClientHandler (зависит от GameSession)
    ↓ использует
GameSessionManager (независимый)
    ↓ управляет
GameSession (независимый)
